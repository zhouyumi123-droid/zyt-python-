# 零基础先看

这份文件是专门写给完全没学过 Python 的你。

如果你以前看到这些词会慌：

- Python
- 代码
- JSON
- 参数
- 报错
- API

很正常。

你现在不需要一下子全部搞懂。

你只需要先建立一个最基本的理解框架。

## 你到底在学什么

你不是在学“高深编程”。

你在学的是：

怎么让电脑按你的要求，把一件具体的小事做完。

比如：

- 整理文字
- 读取资料
- 生成选题
- 把一种格式转换成另一种格式

这就是 AI 自动化工作里最常见的基础动作。

## 为什么你要学 Python

因为像 Coze 这样的工具很强，
但你迟早会遇到这些情况：

- 平台里的节点不够用
- 你想自定义处理内容
- 你想读取本地文件
- 你想批量处理数据
- 你想排查到底哪一步出了问题

这时候，Python 就像你的“万能小扳手”。

## 你现在不用会什么

你现在不用会：

- 算法
- 数学
- 后端开发
- 数据结构
- 面向对象

这些以后再说。

## 你现在先会什么就够了

### 1. 会运行脚本

先把一个脚本跑起来。

### 2. 会看输入和输出

知道脚本吃了什么，吐出了什么。

### 3. 会做很小的修改

比如改一句文字、改一个输出格式。

### 4. 会把报错发出来

会求助，本身就是能力。

## 你接下来会反复遇到的 6 个基础词

### 1. 文件

就是电脑上的一个东西。

比如：

- `.txt`
- `.json`
- `.md`
- `.py`

### 2. 变量

变量可以理解成“一个有名字的盒子”。

盒子里可以装内容。

### 3. 字符串

字符串就是一段文字。

比如：

`"你好"`

### 4. 列表

列表就是“很多东西排成一列”。

比如：

`["苹果", "香蕉", "橙子"]`

### 5. 字典

字典就是“名字 : 对应内容”的组合。

比如：

`{"姓名": "小王", "职业": "运营"}`

### 6. 函数

函数可以理解成：

“一个固定的小动作”

你给它输入，它给你输出。

## 什么是 JSON

你现在可以把 JSON 理解成：

“一种很常见的数据整理格式”

它适合让程序看懂。

先不用背定义，
以后我们会拿真实例子一点点拆。

## 什么是参数

参数就是你运行脚本时，
额外告诉它的信息。

比如：

- 输入文件在哪
- 输出文件放哪

## 什么是报错

报错不是你不行。

报错只是电脑在告诉你：

“我这里没法继续了，原因是这个”

你的任务不是害怕报错，
而是学会看报错、贴报错、修报错。

## 你现在的学习原则

1. 先跑通，再理解
2. 先理解一小块，不求全懂
3. 不追求背知识，追求会操作
4. 每次只前进一步

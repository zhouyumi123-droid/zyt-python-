# 工作流 v1 固定测试用例

## 为什么要固定测试输入

你现在已经不是“随便试试看”阶段了。

从这一步开始，要学会用固定输入反复测试工作流。

这样你才能知道：

- 是提示词变好了
- 还是只是刚好这次结果不错

## 测试方法

每次只改一个节点的提示词。

改完后，用下面 3 组输入全部跑一遍。

跑完后，对比：

- 选题方向是否稳定
- 脚本是否仍然可拍
- 发布包装是否更像平台文案

## 测试组 1：日报机器人

- `zhuti`：`用 Coze 搭一个自动写日报机器人`
- `fabupingtai`：`抖音`
- `mubiaorenqun`：`不会写代码、想转行 AI 自动化的新手`
- `neirongfengge`：`实操演示、结果先行、少空话、不要招聘腔`

重点观察：

- 会不会回到空泛岗位内容
- 封面标题会不会太长
- 标签里有没有 `Coze`、`日报`、`AI自动化`

## 测试组 2：会议纪要助手

- `zhuti`：`用 Coze 搭一个会议纪要整理助手`
- `fabupingtai`：`抖音`
- `mubiaorenqun`：`经常开会、讨厌手动整理纪要的上班族`
- `neirongfengge`：`直接演示、少解释、先给结果`

重点观察：

- 选题会不会具体到“会议录音 -> 整理输出”
- 脚本里有没有明确录屏步骤
- 发布文案会不会过长

## 测试组 3：评论整理助手

- `zhuti`：`用 Coze 搭一个短视频评论整理助手`
- `fabupingtai`：`抖音`
- `mubiaorenqun`：`做内容运营、需要整理用户反馈的人`
- `neirongfengge`：`务实、能落地、像真实工作演示`

重点观察：

- 输出是不是围绕“评论收集 -> 分类 -> 输出”
- 会不会跑偏成泛泛内容运营建议
- 标签是否更贴近运营人群

## 记录模板

每跑一组，至少记 3 个结果：

1. 最佳选题是否合理
2. 脚本是否能直接拍
3. 发布包装是否像真人会发的内容

## 当前通过标准

3 组输入都满足下面条件，就可以认为工作流 v1 基本稳定：

1. 节点都能正常返回结果
2. 没有明显跑偏到行业分析或求职建议
3. 脚本里有可演示步骤
4. 发布包装短、直接、可发

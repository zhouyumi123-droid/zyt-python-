# 大模型4 发布包装优化 v2

## 为什么要优化这个节点

你现在的 `大模型4 发布包装` 已经能出结果。

但当前常见问题有 3 个：

- 封面标题偏平
- 发布文案偏说明书
- 标签不够贴近搜索习惯

所以这个节点的目标不是“能不能跑”，而是：

让输出更像短视频平台上真的会发出去的内容。

## 建议模型使用方式

- 日常调试：优先用更快的模型
- 最终定稿：如果发现语气发散，再切回更稳的模型对比

## 建议系统提示词

把 `大模型4 发布包装` 的系统提示词替换成下面这版：

```text
你是一个短视频发布包装助手。

你的任务：
根据已经生成好的视频脚本，输出适合抖音直接发布的包装内容。

严格要求：
1. 面向抖音发布，不是写教程说明，不是写产品介绍
2. 封面标题必须短、直接、像真人会点开的标题
3. 发布文案控制在 2 到 4 句，不要写成长说明
4. 发布文案语气要像真人发视频，不要像培训资料，不要像公文
5. 可以有一点口语感，但不要浮夸，不要喊口号
6. 话题标签只给 5 个
7. 标签优先使用用户可能会搜索的词，比如工具名、任务名、人群词、学习词
8. 不要输出 JSON
9. 不要输出格式外内容

请严格按以下格式输出：

封面标题：
发布文案：
话题标签：
1.
2.
3.
4.
5.
```

## 建议用户提示词

用户提示词替换成下面这版：

```text
发布平台：{{fabupingtai}}

下面是一条已经写好的短视频脚本，请输出可以直接用于发布的包装内容：

{{video_script}}
```

## 你要重点看什么

每次跑完后，不要只看“有没有结果”，而是看下面 3 点：

### 1. 封面标题

是否满足：

- 一眼能看懂
- 不绕
- 不像功能说明

如果标题太长，说明模型还在“解释”，没有进入“发布包装”状态。

### 2. 发布文案

是否满足：

- 像真人会写的话
- 不像文档摘要
- 读完后愿意继续看视频

如果文案一大段都在讲功能原理，就说明提示词还不够硬。

### 3. 话题标签

是否满足：

- 有工具词：如 `Coze`
- 有任务词：如 `日报`
- 有人群词：如 `AI新手`
- 有学习词：如 `AI自动化`

如果标签全是很泛的名词，后面可以再继续收紧。

## 当前更推荐的标签方向

和你现在的目标更贴近的标签类型：

- `Coze教程`
- `AI自动化`
- `日报自动化`
- `AI新手`
- `不会代码也能做`

不是每次都要一模一样，但方向应该尽量往这边靠。

## 什么时候算通过

这个节点暂时达到下面标准，就算通过：

1. 连续 3 组测试输入都能正常输出
2. 封面标题不再像说明书
3. 发布文案控制在短内容范围内
4. 标签里至少有工具词、任务词、人群词中的 2 类

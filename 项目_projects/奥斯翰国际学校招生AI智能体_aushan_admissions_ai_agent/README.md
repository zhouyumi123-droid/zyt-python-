# 奥斯翰国际学校招生 AI 智能体项目

## 这是什么

这是你当前最重要的真实业务项目。

目标不是做一个泛泛的“内容助手”，而是做一套真正服务于国际学校线上招生的 AI 智能体体系。

当前业务背景：

- 你负责奥斯翰国际学校线上招生板块
- 板块包含直播和短视频内容
- 直播侧已经有 Codex 工作台和飞书自动化工作流
- 短视频侧需要大量基于学校特色和国际课程优势的内容批量产出

所以，这个项目的核心任务是：

用 Coze 和 Codex 配合，搭出一套可跑通、可迭代、可复制的招生 AI 内容智能体。

## 这个项目最终想解决什么

先解决 3 个现实问题：

1. 学校资料很多，但内容生产不够快
2. 招生内容需要持续更新，人工写稿压力大
3. 不同内容形态很多，视频、宣讲、数字人脚本都需要批量化

## 这个项目以后可能长成什么

如果奥斯翰国际学校这个试点跑通，后面可以往两个方向走：

### 1. 行业标准化智能体

面向学校、教育机构等一类客户，做相对标准化的低客单价产品。

### 2. 企业定制化智能体

面向有更复杂需求的学校或其他行业客户，做高客单价定制方案。

## 当前最重要的原则

现在不要一上来就做“大而全”的智能体。

先做一个能真正帮你当前工作省时间、提效率、能展示结果的最小版本。

## 第一阶段只做 4 类内容能力

### 1. 学校资料理解

输入学校资料，输出结构化摘要。

目标：

- 能快速提炼学校卖点
- 能区分课程优势、升学优势、校园特色、师资特色

### 2. 招生短视频选题与脚本

输入招生主题，输出：

- 选题方向
- 视频标题
- 视频脚本
- 发布包装

### 3. 宣讲视频与介绍视频脚本

把同一份资料，改写成：

- 宣讲口播稿
- 校长/老师介绍稿
- 家长沟通视频稿

### 4. 数字人批量脚本

针对数字人出镜场景，输出更短、更规整、更稳定的脚本版本。

## 这个项目推荐的建设顺序

### 阶段 A：资料层

先把学校资料整理成能喂给智能体的结构。

现在最先需要的不是更复杂的提示词，
而是清楚知道：

- 你手里有哪些资料
- 哪些资料能公开讲
- 哪些资料适合做视频
- 哪些资料需要校对

### 阶段 B：内容层

先做“短视频内容智能体 v1”。

它负责：

- 选题
- 脚本
- 发布包装

### 阶段 C：招生问答层

再做“学校问答智能体”。

它负责：

- 学校介绍问答
- 课程体系问答
- 招生政策问答
- 家长常见问题问答

### 阶段 D：批量化层

最后再做批量产出：

- 一份资料出多条脚本
- 一条脚本改多种视频形态
- 一种主题出多个年级/课程版本

## 你当前最该先拿下的一个最小成果

不是“做完整个招生智能体平台”。

而是先做出这个：

**奥斯翰国际学校短视频招生内容智能体 v1**

它最少要做到：

1. 输入学校资料主题
2. 输出 5 个视频选题
3. 选 1 个最适合先做的
4. 生成可拍脚本
5. 生成发布包装

## 这个项目和你的 FDE 储备是什么关系

这个项目本身就是你的训练场。

你在这里练到的，不只是内容生产，而是更底层的能力：

- 把模糊需求拆成系统
- 把系统拆成节点
- 把节点变成可运行工作流
- 把工作流变成可交付产品

这正是你后面做 FDE 储备、作品集储备、商业化储备时最值钱的部分。
